# キャスティング部 指示書

## 役割定義

キャスティング部は Noah Axice における **人脈の在庫管理・キャスティング・人材開拓** を担う専門部署。

Noah Axice の商品の原材料は「人脈」。  
キャスティング部はその**調達・管理・運用**を担う。

---

## 管理ファイル一覧

| ファイル | 役割 |
|---|---|
| `人脈台帳.csv` | 全人脈のマスターデータ。在庫リスト本体 |
| `スキルタグ一覧.csv` | タグのマスター。キャスティング検索の基準 |
| `人材探索リスト.csv` | 不足人材の補充依頼・探索状況管理 |
| `アサイン履歴.csv` | プロジェクトへのアサイン実績・評価記録 |

## 人脈登録ルール

### IDの採番

- 形式：`NA-XXX`（例：NA-001, NA-002）
- 台帳の最終IDを確認して連番で採番すること

### 登録のトリガー

会長（あつき）がチャットで「〇〇さんを登録して」「こういう人と繋がった」と伝えてきたら即座に登録する。

### 登録時に必須で記録する項目

1. **ID**（連番）
2. **氏名**
3. **スキルタグ**（スキルタグ一覧.csvから選択。ない場合は新規追加）
4. **関係の強さ**（1-5）
5. **稼働状況**
6. **登録日・更新日**

### 任意でも可能な限り記録すること

- 出会った経緯
- 人柄・特記事項
- 連絡手段・連絡先
- Noah Axice関心度

## キャスティング実行ルール

### キャスティング依頼のトリガー

「〇〇ができる人いる？」「〇〇プロジェクトに誰を入れたらいい？」と聞かれたら実行する。

### キャスティング手順

1. `人脈台帳.csv` を読み込む
2. 依頼内容に合うスキルタグで絞り込む
3. 以下の優先順位で候補を並べる：
   - 関係の強さが高い（5→1）
   - 稼働状況が「フリー」または「副業可」
   - Noah Axice関心度が高い
4. 候補を3〜5名提示し、理由を添える
5. 必要に応じて「この人を使う場合の注意点」も提示する

### キャスティング後

アサイン確定したら `アサイン履歴.csv` に記録する。

## 人材探索（在庫補充）ルール

### 探索依頼のトリガー

「〇〇スキルの人が足りない」「新しく〇〇を探したい」と言われたら実行する。

### 対応手順

1. `人材探索リスト.csv` に探索依頼を追記する
2. 台帳内に近いスキルを持つ人がいれば提示（紹介してもらえる可能性）
3. 会長に「どういうルートで探すか」を提案する
   - 例：既存人脈からの紹介、SNS検索、ダンスイベントでのスカウト等

---

## 引き継ぎガイド（正社員向け）

このフォルダを引き継いだ担当者へ：

1. **人脈台帳.csv**が在庫リスト本体。定期的に更新すること
2. 最終コンタクト日が90日以上空いている人には定期フォローを入れること
3. スキルタグは`スキルタグ一覧.csv`に統一。独自タグを勝手に作らない
4. プロジェクトアサイン後は必ず`アサイン履歴.csv`に記録し、評価も入れること
5. 関係の強さ・Noah Axice関心度は会長に確認しながら更新すること

## 関係の強さ 定義

| レベル | 意味 |
|---|---|
| 5 | 深い信頼関係。仕事を一緒にやれる |
| 4 | 友人レベル。連絡すれば動いてくれる |
| 3 | 知り合い。声をかければ話を聞いてくれる |
| 2 | 名前と顔が一致する程度 |
| 1 | 繋がりはあるが面識薄い |

## 稼働状況 定義

| ステータス | 意味 |
|---|---|
| フリー | 現在メイン案件なし・すぐ動ける |
| 副業可 | メイン仕事あり・副業・スポット参加可能 |
| 多忙 | 現在は動けない状況 |
| 不明 | 確認できていない |
